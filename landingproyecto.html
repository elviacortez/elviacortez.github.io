Recopilación de datos mediante API REST

Se trabajó con los datos de lanzamiento de SpaceX recopilados a través de una API, específicamente la API REST de SpaceX. Esta API proporciona datos sobre los lanzamientos, incluida información sobre el cohete utilizado, la carga útil entregada, las especificaciones del lanzamiento, las especificaciones de aterrizaje y el resultado del aterrizaje. El objetivo es utilizar estos datos para predecir si SpaceX intentará aterrizar un cohete o no. Se realiza una solicitud de obtención utilizando la biblioteca de solicitudes para obtener los datos de lanzamiento, y se utiliza para obtener los datos de la API. Este resultado se puede ver llamando al método .json (). La respuesta tendrá la forma de un JSON, específicamente una lista de objetos JSON. En concreto, se tiene una lista de objetos JSON, cada uno de los cuales representa un lanzamiento. Para convertir este JSON en un marco de datos, se usó la función json_normalize. Esta función permite «normalizar» los datos json estructurados en una tabla plana. Columnas de datos obtenidas de la API de Space X: Número de vuelo, fecha, versión del booster, masa de la carga útil, órbita, lugar de lanzamiento, resultado, ights, GridFins, reutilizados, patas, pista de aterrizaje, bloque, recuento de reutilizados, serie, longitud, latitud

Disputa de datos

La intención es transformar los datos sin procesar en un conjunto de datos limpio que proporcione datos significativos sobre la situación que estamos intentando abordar: ordenar los datos mediante una API,muestrear los datos y tratar los valores nulos. En algunas columnas, como Rocket, se tenía un número de identificación, no datos reales, esto significó que se tuvo que volver a utilizar la API para dirigirse a otro punto final a fin de recopilar datos específicos para cada número de identificación. Con las funciones ya creadas se usaron los siguientes: Booster, Launchpad, payload y core. Los datos se almacenaron en listas y se utilizaron para crear el conjunto de datos. Otro problema que hubo es que los datos de lanzamiento que se tenían incluían datos del propulsor Falcon 1, mientras que solo se querías el Falcon 9. En esta etapa se analizaron cada uno de los atributos y se transformaron algunos de ellos. Los de lanzamiento, la columna de órbitas, la altitud, la duración de cada una de estas órbitas, el atributo oucomes, que determina si la primera etapa del lanzamiento tuvo éxito o no, fueron analizados. Se puso de manifiesto que hay 8 tipos que indican si el booster aterrizó con éxito o no. Este atributo fue convertido a las clases 0 y 1 y el valor fue retenido en una variable, esencial para la búsqueda. Del mismo modo, se determinaron qué atributos están relacionados con los aterrizajes con éxito y se corvirtieron las variables categóricas, preparando los datos para un modelo de aprendizaje automático. Se creó una etiqueta de entrenamiento con resultados de aterrizaje donde éxito = 1 y fracaso = 0. La columna de resultados tiene dos componentes "Resultado de la misión" "Lugar de aterrizaje" Se añadió una nueva columna de etiqueta de entrenamiento "clase" con un valor de 1 si "Resultado de la misión" es Verdadero y 0 en caso contrario. Clase: Verdadero ASDS, Verdadero RTLS y Verdadero Océano : 1 Clase: Ninguno Ninguno, ASDS Falso, ASDS Ninguno, Océano Falso, RTLS Falso: 0

Análisis predictivo (clasificación)

Se crea un proceso de aprendizaje automático para predecir si la primera etapa aterrizará de acuerdo con los datos de prácticas anteriores, para ello se siguieron estos pasos: 1- Cargar los datos como un marco de datos de pandas para explorar los datos y encontrar las etiquetas de entrenamiento, para realizar esta tarea se creó una matriz NumPy a partir de la columna Clase en datos, aplicando el método to_numpy() y luego asignado a la variable. -Estandarizar los datos en X y luego reasignarlos a la variable X usando la transformación. - Utilizar la función train_test_split para dividir los datos X e Y en datos de entrenamiento y prueba. Establecer el parámetro test_size a 0,2 y random_state a 2. 2- Para encontrar el método que mejor convenga: - Crear un objeto GridSearchCV y asignar un conjunto de parámetros. - Cada uno de los objetos fue creado con cv=10 y equipado con los datos de entrenamiento para encontrar el mejor modelo -Los cuatro modelos, regresión logística, SVM, el árbol de decisión y el KNN, tienen la misma puntuación, por lo que todos se desempeñan con una precisión igual al 83,3% y todos son buenos en predecir correctamente los casos positivos, pero vale la pena señalar que el tamaño del ensayo es pequeño, con un tamaño de muestra de solo de 18. Es necesario utilizar más datos para determinar el mejor modelo.

Conclusiones

El análisis mostró que existe una correlación positiva entre el número de vuelos y la tasa de éxito, cuanto mayor es el número de vuelos mayor es el éxito y se ha reflejado que el promedio de la tasa de éxito es del 66,6%, ha mejorado con el paso de los años. Hay determinadas órbitas como SSO, HEO, GEO y ES-L1 donde los lanzamientos tuvieron más éxito, al ser órbitas geosincrónicas, pueden permanecer en una posición fija y son ideales para operaciones de satélites de comunicaciones, pueden estar más cerca del ecuador. La tasa de éxito puede vincularse a la masa de la carga útil, ya que las cargas útiles más ligeras generalmente eran más exitosas que las más pesadas. Los sitios de lanzamiento están ubicados estratégicamente cerca de algunas carreteras y vías férreas para el transporte de personal y carga, lejos de las ciudades por razones de seguridad y cerca de la costa en caso de lanzamientos fallidos. - SpaceY puede utilizar los modelos para predecir con una precisión relativamente alta si un aterrizaje de etapa 1 tendrá éxito. Si es posible, se deben recopilar más datos para determinar el mejor modelo de aprendizaje automático y mejorar la precisión, porque los datos son insuficientes, y puede suceder que hubo poca variación en las clases, sobreajuste o datos homogéneos.

Consideraciones finales

Según los resultados predichos por los modelos de clasificación, existe un 83% de probabilidad de que SpaceX pueda reutilizar la primera etapa de SpaceX y si los lanzamientos logran aterrizar con éxito, se puede predecir que cada lanzamiento costaría 62 millones de dólares, aunque todavía cueste un poco más, es una buena idea de negocio para lanzar cohetes para diversas operaciones, especialmente para satélites de comunicaciones, porque según el EDA, la mayor tasa de éxito se produce en las órbitas más cercanas al ecuador, que son geosincrónicos y pueden permanecer en un sitio fijo desde la posición de la Tierra. Teniendo en cuenta que la carga útil en cada lanzamiento debe estar entre 2000 y 4000 Kg para garantizar el éxito y ubicar el sitio de lanzamiento en KSC LC 39A o CCAF SLC 40, que si bien han tenido algunos lanzamientos fallidos a medida que aumentan los vuelos, las condiciones mejorarán, demostrando tener una tasa de éxito promedio del 66%. En términos generales, es bastante factible considerar el lanzamiento de cohetes en los próximos años, siguiendo estas pautas para ofrecerlos a muy buen precio.

Ideas generales

Una de las ideas innovadoras es reutilizar el cohete en cada lanzamiento, así como los aviones con vuelos internacionales, pero además de eso, un cohete que no utilice combustible, sino más bien un componente más accesible y menos perjudicial para el medio ambiente, podría ser agua mezclada con otro componente químico que desarrolle altas velocidades y consuma menos energía. El diseño de los cohetes también es de suma importancia para lograr velocidad y romper la fuerza de resistencia encontrada al salir de la atmósfera, con un estilo casa, pero con una forma aerodinámica que proteja a las personas de los riesgos del exterior. Aunque ya se está estudiando el lanzamiento horizontal de cohetes, también está bien considerar las plataformas de lanzamiento móviles, en lugar de estar fijas en el suelo, Sería como construir una pala excavadora gigante, de las que se utilizan en la construcción, con una base redonda o alargada para colocar el cohete y mantenerlo en la mejor posición para el lanzamiento, y lo mejor es que sería móvil y se podría conducir a cualquier lugar con la base giratoria, para la mejor ubicación y posición. Para iniciar este proyecto de una manera más comercial, sería más conveniente comenzar con lanzamientos a las órbitas más cercanas a la Tierra, que según el éxito es mucho mayor y con poca carga, para familiarizarse con este tipo de vuelos.
