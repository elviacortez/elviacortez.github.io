CRISP-DM

Es un modelo cíclico y muy flexible. Se requiere flexibilidad en cada paso junto con comunicación para mantener el proyecto encaminado. En cualquiera de las seis etapas, puede ser necesario revisar una etapa anterior y realizar cambios. El punto clave de este proceso es que es cíclico; Por lo tanto, incluso al final tendrá otro encuentro de entendimiento empresarial para discutir la viabilidad después de la implementación.Es un proceso altamente iterativo y nunca termina.
Es un método probado para orientar sus trabajos de minería de datos.
Como metodología, incluye descripciones de las fases normales de un proyecto, las tareas necesarias en cada fase y una explicación de las relaciones entre las tareas.
Como modelo de proceso,CRISP-DM ofrece un resumen del ciclo vital de minería de datos.
CRISP-DM, son las siglas de Cross-Industry Standard Process for Data Mining.

Del problema al enfoque

Esta etapa es la más importante porque es donde se describe la intención del proyecto. La metodología fundamental y CRISP-DM están alineadas aquí. Requiere comunicación y claridad. La dificultad aquí es que las partes interesadas tienen diferentes objetivos, sesgos y modalidades de relacionar la información. No todos ven las mismas cosas ni de la misma manera. Sin una perspectiva clara, concisa y completa de cuáles son los objetivos del proyecto, los recursos se gastarán innecesariamente.
Es importante reconocer:
-La necesidad de comprender y priorizar el objetivo empresarial.
-La forma en que el apoyo de las partes interesadas influye en un proyecto.
-La importancia de seleccionar el modelo adecuado.
-Cuándo utilizar un modelo predictivo, descriptivo o de clasificación.
-La elección del modelo depende del objetivo del análisis de datos. Los modelos descriptivos se utilizan para entender los datos, los modelos predictivos se utilizan para predecir resultados futuros y los modelos de clasificación se utilizan para categorizar objetos o individuos.
Los científicos de datos pueden utilizar también un enfoque "descendente" o un enfoque "ascendente" para la ciencia de datos. Estos dos enfoques se refieren a:
-Enfoque "descendente" - primero se define un problema empresarial y luego se analizan los datos para encontrar una solución.
-Enfoque "ascendente" - empezar con los datos y luego plantear un problema empresarial basado en los datos.

De los requisitos a la recopilación

Basándose en la comprensión del problema en cuestión, y luego utilizando el enfoque analítico seleccionado, el científico de datos está listo para comenzar.
Antes de emprender las etapas de recopilación y preparación de datos de la metodología, es vital definir los requisitos de datos para la utilización del modelo seleccionado. La fase de requisitos de datos de la metodología de la ciencia de datos consiste en identificar el contenido, los formatos y las fuentes de datos necesarios para la recopilación inicial de datos.
En la fase de recopilación de datos, se revisan las necesidades de datos y se decide si se necesitan más datos o no.
Los científicos de datos determinan cómo recopilar los datos, identifican los datos necesarios para el modelado de datos y determinan cómo preparar los datos.
Se puede pensar en estas etapas como en una tarea de cocina, en la que el problema en cuestión es una receta, y los datos para responder a la pregunta son los ingredientes.
En la fase de recopilación de datos, pueden aplicarse al conjunto de datos, técnicas como la estadística descriptiva y visualización, para evaluar el contenido, la calidad y las ideas iniciales sobre los datos.
Es importante saber en esta etapa:
-La importancia de definir los requisitos de datos para su modelo.
-Por qué importan el contenido, el formato y la representación de sus datos.
-La importancia de identificar las fuentes de datos correctas para su proyecto.
-Cómo manejar los datos no disponibles y redundantes.
-Anticiparse a las necesidades de las etapas futuras del proceso.

De la comprensión a la preparación de datos

Junto con la recolección de datos y la comprensión de datos, la preparación de datos es la fase más dispendiosa de un proyecto de ciencia de datos, ocupando por lo general el setenta por ciento y aún el noventa por ciento del tiempo total del proyecto.
Automatizar algunos procesos de recolección y preparación de datos en la base datos puede reducir este tiempo a tan sólo el 50 por ciento. Este ahorro se traduce en más tiempo para que los científicos de datos se centren en crear modelos.
La comprensión del negocio es una etapa importante en la metodología de la ciencia de datos porque define claramente el problema y las necesidades desde una perspectiva empresarial.
En esta fase se formatean correctamente los datos y se toman en cuenta las siguientes consideraciones:
-La importancia de las estadísticas descriptivas.
-Cómo gestionar los datos ausentes, no válidos o engañosos.
-La necesidad de limpiar los datos y, a veces, de transformarlos.
-Las consecuencias de los datos erróneos para el modelo.
-La comprensión de los datos es iterativa; se aprende más sobre los datos cuanto más se estudian.

Del modelado a la evaluación

El modelado es la etapa de la metodología de la ciencia de datos, en la que el científico de datos tiene la oportunidad de probar la salsa y determinar si es buena o si necesita más condimentos, si lo comparamos con una receta de cocina.
Esta parte está orientada a responder a dos preguntas claves: en primer lugar, ¿cuál es el propósito del modelado de datos y, en segundo lugar, cuáles son algunas de las características de este proceso? El modelado de datos se centra en el desarrollo de modelos descriptivos o predictivos.

Como científico de datos, la exploración de datos (EDA) es el tipo de análisis que utiliza con:
Estadísticas descriptivas que tienen tres categorias:
-Distribución de frecuencias
-Medidas de tendencia central(la media, la mediana y la moda,) y
-Medidas de variabilidad (la desviación estándar o varianza, los valores mínimo y máximo de las variables, entre otras)

y técnicas de visualización de datos (como gráficos y mapas).

Un ejemplo de modelo descriptivo podría examinar aspectos como: si una persona hizo esto, es probable que prefiera aquello. Describe un fenómeno, situación o población en particular. Su objetivo principal es recopilar y analizar datos cuantificables para obtener una comprensión más profunda del fenómeno en estudio. Este tipo de investigación se centra en responder a preguntas como "¿Qué ha pasado?", "¿Cuántos?", y "¿Cuáles?, utilizando técnicas como la observación, la encuesta, entre otras.

Un modelo predictivo intenta arrojar resultados de tipo sí/no o parar/continuar.Estos modelos se basan en el enfoque analítico que se adoptó, ya sea basado en las estadísticas o en el aprendizaje automático.Utiliza datos históricos y actuales para hacer predicciones sobre el futuro. Este tipo de modelos se basa en técnicas matemáticas, estadísticas y de machine learning avanzadas para analizar los datos y determinar y extrapolar las tendencias ocultas.

Es importante señalar que la exactitud y utilidad de los resultados de un modelo predictivo dependen del nivel de análisis de los datos. También el científico de datos utilizará un conjunto de entrenamiento para el modelado predictivo.
Un conjunto de entrenamiento es un conjunto de datos históricos en los que ya se conocen los resultados. El conjunto de entrenamiento actúa como un indicador para determinar si es necesario calibrar el modelo.

En esta etapa, el científico de datos probará con diferentes algoritmos para asegurarse de que las variables en juego sean realmente necesarias.
El éxito de la recopilación, preparación y modelización de los datos depende de la comprensión del problema en cuestión y del enfoque analítico adecuado que se adopte. Los datos respaldan la respuesta a la pregunta y, al igual que la calidad de los ingredientes utilizados en la cocina, preparan el terreno para el resultado.
Es necesario afinar, ajustar y ajustar constantemente cada paso para garantizar que el resultado sea sólido. En la metodología descriptiva de ciencia de datos de John Rollins, el marco está diseñado para hacer tres cosas:

-Primero, entender la pregunta en cuestión.
-En segundo lugar, seleccionar un enfoque o método analítico para resolver el problema y
-Tercer lugar, obtener, comprender, preparar y modelar los datos.

El objetivo final es llevar al científico de datos a un punto en el que se pueda crear un modelo de datos para responder a la pregunta. Con la cena a punto de servirse y con un invitado hambriento en la mesa, la pregunta clave es: ¿he preparado lo suficiente para comer? Bueno, esperemos que sí.

La evaluación proporciona una valoración de la calidad del modelo, pero también de la solicitud inicial. ¿El modelo utilizado realmente responde a la primera pregunta?
¿Responde o hay que ajustarlo?

La evaluación del modelo puede tener dos etapas principales.

La primera se utiliza para garantizar que el modelo funcione según lo previsto, es la etapa de las medidas de diagnóstico. Si el modelo es predictivo, verificar si la respuesta que puede dar el modelo es compatible con el diseño inicial. Se puede utilizar un árbol de decisión para evaluar, para ver dónde hay áreas que necesitan ajuste.

La segunda etapa de evaluación que se puede utilizar es la prueba de significación estadística. Este tipo de evaluación garantiza que los datos se integren adecuadamente dentro del modelo. Se puede aplicar al modelo para garantizar que se procese e interprete. Aunque no siempre se aplican, depende del contexto
-La evaluación del modelo incluye asegurarse de que el modelo funciona según lo previsto.

La curva ROC es una representación gráfica del rendimiento de un modelo de clasificación binaria para todos los umbrales de clasificación.
ROC son las siglas de Receiver Operating Characteristic curve (curva característica operativa del receptor), que se desarrolló originalmente para detectar aviones enemigos en los radares.

Para concluir, las etapas finales de la metodología de la ciencia de datos son un ciclo iterativo entre: Modelización, evaluación, despliegue y retroalimentación.

Lo importante en esta etapa es reconocer:
-La diferencia entre modelos descriptivos y predictivos.
-El papel de los conjuntos de entrenamiento y los conjuntos de prueba.
-La importancia de preguntarse si se ha respondido a la pregunta.
-Por qué son necesarias las herramientas de medidas de diagnóstico.
-El propósito de las pruebas de significación estadística.
-Que la modelización y la evaluación son procesos iterativos.

Del despliegue a la retroalimentación

Cuando el modelo ha sido evaluado y el científico de datos confía en que funcionará, se despliega y se somete a la prueba definitiva: el verdadero uso de campo en tiempo real. La retroalimentación de los usuarios ayudará a refinar el modelo y evaluar su desempeño e impacto. Si bien un modelo de ciencia de datos proporcionará una respuesta, la respuesta consiste en abordar la primera pregunta. La clave para que sea relevante y útil es que las partes interesadas se familiarizen con el vehículo. En un escenario empresarial, las partes interesadas incluyen al propietario de la solución, al marketing y a los desarrolladores de aplicaciones y tienen diferentes conocimientos, como gestión de TI, para ayudar a que esto suceda. Una vez que se evalúa el modelo y el científico de datos está seguro de que funcionará, se implementa y es puesto a prueba final. Dependiendo del propósito del modelo, el resultado se puede aplicar en cualquier lugar. Puede estar disponible para un grupo limitado de usuarios o en un entorno de prueba para generar confianza.
La retroalimentación es esencial para la viabilidad a largo plazo del modelo.
Si un científico de datos determina que construir un sistema de recomendación es la solución para un problema empresarial concreto que tiene entre manos, esto está representado en esta etapa

¿Cuáles son las tres razones importantes por las que los científicos de datos deben mantener una comunicación continua con los patrocinadores empresariales a lo largo de un proyecto?
-Para que los patrocinadores empresariales puedan revisar los resultados intermedios.
-De modo que los patrocinadores empresariales puedan asegurarse de que el trabajo sigue por buen camino para generar la solución prevista.
-Para que los patrocinadores empresariales puedan aportar su experiencia en el sector.
El despliegue de un modelo en producción representa el inicio de un proceso iterativo que va desde la retroalimentación, pasando por el perfeccionamiento del modelo, hasta llegar a la redistribución
Es de hacer notar:
-La importancia de la aportación de las partes interesadas.
-La escala de despliegue.
-La importancia de incorporar la retroalimentación para refinar el modelo.
-El modelo perfeccionado debe volver a desplegarse.
-Este proceso debe repetirse tantas veces como sea necesario.
